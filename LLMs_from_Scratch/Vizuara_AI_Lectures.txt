Lecture 8: GPT Tokenizer (BYTE-PAIR ENCODING)  (Tokenizer.ipynb notebook) ----> WATCH THE LECTURE VIDEO AGAIN IF CONFUSED

* Tokenization Algorithms:

- Words based tokenizer --> it has limitations because it does not account for words (tokens) not in the dataset. Also, semantic meaning of words is lost (e.g. boy != boys in meaning)

- Character based tokenizer --> splits every character into tokens and assign a token ID to each character. Limitations is: the meaning associated with words is completely lost and the tokenized sequence is much longer than the initial raw text.

- Sub-words based tokenizer --> Used by GPT byte tokenizer. It has 2 major rules:
1. Do not split frequently used words into smaller sub-words. 
2. Split the rare words into smaller meaningful sub-words.
3. Example: boy and boys (boy and boys should not be split if frequent in the dataset; but if necessary, it should be split into characters (boys == boy, s)

- Advantages of Sub-words based Tokenization:
1. Sub-words retain the root-word meaning of words. e.g. token, tokens, tokenizer with token as the root word
2. It helps the model to understand that TOKENIZATION AND MODERNIZATION have different root words but the same suffix (ization)

* BYTE Pair Encoding Algorithm (BPE) is a sub-word Tokenization Algorithm: 

- If a dataset contains common byte pairs, replace the byte pairs with a byte not present in the data  eg. (aaadbaadb), aa can be replace with a byte(letter not present == (ZabdZdb) == (ZaYZY)

- Common words in a vocabulary are represented as a single token.  (similar to Sub-words Based Tokenizer)

- Rare words are broken down into two or more sub-words token.

Lecture 11: Positional Embeddings Encoding

* Positional Vectors have the same input as the vector embeddings

* 2 types of positional embeddings:

- Absolute  --> vector embeddings of token + position of token i.e. (Input Embeddings = Vector Embeddings + Position of Vector Embeddings)
		fixed order of tokens. e.g. sequence generation.

- Relative  --> deals with how far apart (distance) of vector embeddings, rather than the exact position. The advantage is model can generalize better to sequence of varying length, even if it has not seen such length during training.

- Both Absolute and Relative Positional Encodings allow the LLM to understand the order and relationship between tokens, ensuring accurate and context-aware predictions

- The choice between the 2 depends on the nature of the data being processed.

- OPENAI GPTs used absolute positional encoding

- During model training, the weighs are optimized for vector embeddings and positional embeddings

Lecture 13: Introduction to the Attention Mechanisms in Large Language Model

- Before the advent of transformer architecture and GPTs (which use Attention Mechanism), the Recurrent Neural Network (RNNS) were the most popular encoder-decoder architecture that was used for language translation

- Output from the previous text is fed as input to the current text.

* How the Recurrent Neural Network works:

- RNN receives input texts

- The input text is passed to an Encoder (processes input text sequentially)

- Updates Hidden state at each step (Hidden state is like a memory with internal values as hidden layers). It's always updated whenever a new text is received as input
(Hidden states are like the hidden layers of a Neural network)

- Final Hidden state --> encoder captures sentence meaning from every text and also the context

- Decoder takes the final hidden state to generate translated sentence one word at a time. Decoder also updates its hidden state at each time

- The limitation of RNNs is: RNN cannot directly access previous hidden states (previous words) from the encoder, during the decoding phase. It relies on the current hidden state, which leads to loss of context if the sentence is too long. This limitation hindered RNN architecture is not required to build deep neural network for natural language processing, which gave rise to the transformer architecture with a self-attention mechanism, inspired by Bahdanau (German PRof)

- In the self-attention, the decoder has access to all hidden states (memory of hidden layers) from the encoder, so it pays attention to the most important word for context purposes.

* 4 types of attention Mechanism:

1. Simplified Self Attention: 

2. Self Attention

3. Causal Attention

4. Multi-Head Attention

Lecture 14: Simplified Self Attention Mechanism (No trainable Weights): Coded from Scratch with Python (Attention mechanism.ipynb notebook)

* Goal of Attention Mechanism: 

- After getting the vector embeddings for every tokens in a sentence, get the context vectors for each embedding vector, which determines how important a word is to every other word in the sentence. The context vectors will be inputs to the LLMs.

- In a sentence, take the embedding vector of the most important token (aka query vector), convert the embedding vector to a context vector (which will contain information about the particular query vector and other input vectors/elements)  -----> Attention Score is the solution to this

* Attention Score: 

- computes the importance of a specific query (vector) in relation to other tokens (vectors). 

- This is the first step to getting the context vector

- Dot Product between query vector and other input vectors helps with attention score to determine how align (parallel when θ == 0) or how far (perpendicular when θ == 90) the vectors are.

- Mathematically, dot-product ==> a.b = |a||b|cos θ

- For Parallel/similar vectors, θ = 0 ---> a.b = |a||b|cos 0∘ ---> a.b = |a||b| * 1 == 1.

- For Perpendicular vectors, θ = 90∘ ---> a.b = |a||b|cos (90∘) ---> a.b = |a||b| * 0 == 0.

- In the context of self attention mechanisms, dot product determines which element of a sequence attend to one another. Higher dot product == higher attention score and similarity between   two elements(vectors)

- For more info, look at the Attention mechanism.ipynb notebook

* Attention Weight Calculation (Normalized Attention Score):

- We use Softmax to ensure the extreme large weights are between 0s and 1s. To implement SoftMax, we take the sum of the each weight, divide the total(sum) by the exponential of each individual weight. In Pytorch, we do the same, but the difference is the highest weight is subtracted from the exponential of each weight. This process is done to avoid numerical instability when dealing with large values during training. 

- Attention Weight must sum up to one.

* Context Vector:

- After computing the attention weights, we calculate the context vector.

- Multiply individual embedding vector by individual attention weight and Sum the above resulting multiplied vectors together to get the Context vector for a particular input token (Matrix Vector Multiplication) ----> you will need do this for every input to get the context vector

Lecture 15: Self Attention Mechanism with Trainable Weights - Coded from Scratch with Python (Attention mechanism.ipynb notebook)

* Key Concept: Query, Key and Value Trainable Weight Matrices

* Goal:  

- To pay more attention to words/vector that are contextually useful to a query vector even if they are not semantically related to the query vector.

- This mode of Self Attention Mechanism is used in original transformer architecture, GPT models and most popular LLMs.

- This mode of attention mechanism is also called Scaled Dot-Product Attention.

- The Trainable weight matrices are updated during model training.

- The Trainable weights are crucial because the trained model learns to produce the good context vectors (Enriched embedding Vector). 

- We convert the input embeddings into 3-trainable weight matrices vectors --- Query Weight (Wq), Key Weight (Wk), Vector Weight (Wv)

- Multiply the input embeddings by each of the trainable weight matrices (Wq, Wk, Wv) to get Query, Key and Values Matrices (Vectors). Each row of the Query, Key and Values matrices represent each input. This is known as transformation of the input embeddings into Query Key and Values Vectors.

- The parameters of the 3 trainable weights (Wq, Wk, Wv) matrices are not fixed, and are optimized during model training.

- Check the notebook for the codes to get the Query, Key and Value Vectors, then get the attention scores.

- Attention Scores Calculation = Query vector of a particular input * Keys of all other inputs

Lecture 16: Causal Self Attention Mechanism - Coded from Scratch with Python (Attention mechanism.ipynb notebook)

- Causal Attention is also known as masked attention (a special form of self attention mechanism)

- It restricts the model to only consider previous and current inputs in a sequence, when processing any given token.

- Causal attention contrasts Self Attention mechanism in that Slef Attention allows attention/access to the entire input sequence at once.

- The attention scores in Causal attention ensures that the model only factors in the tokens that occur at or before the current token in the sequence.

* Causal Attention Procedures:

- Get Attention Scores (unnormalized) 

- Get Attention Weights (normalized) 

- Mask Attention Scores above diagonal with zero (unnormalized) 

- Mask Attention Weights (normalized) 

Summary: get the attention weights and zero out the diagonal  (check notebook for more explanation of codes)

Lecture 18: Multi-Head Attention (Weights Split) Part 2 - entire Mathematics Explained 

* Goal:

- Instead of using 2 different weights matrices for the Query matrix, Key Matrix and Vector Matrix as done in lecture 17, we use one Query Weight Matrix to multply the vector embeddings, and split the resulting Queries into different queries. This is because in GPTs, they have up to 96 attention heads and it's unreasonable to have 96 different weights matrices for query, key and values. Hence, we perform weights splits for Query matrix, Key matrix and Value Matrix in Multi-Head Attention Mechanism, to split the heads.

- Mathematically: Head_dimension = d_out / n_heads  ===> d_out = columns of final context vectors, n_heads = number of Heads for (Causal Attention Mechanism)

- Wq, Wk, Wv = d_in * d_out  ----> d_in (number of embedding vector in a dimensional space) == d_out (as used in training GPTs models)

* Procedures for the Multi-Head Attention Class Code in Attention mechanism.ipynb notebook:

1. Get the batch size, number of tokens (num_tokens) and input dimension (d_in) for each vector/input embedding. In the lecture batch_size = 1, num_tokens = 3 for 3 inputs words (The Cat Sleeps), d_in = 6 (six vector embeddings for each word in the vector space.  Mathematically, (batch_size X num_tokens X d_in = 1 X 3 X 6 matrix)

2. Decide output dimesnions (d_out) and number of attention heads (num_heads). In the lecture d_out = 6 = d_in, num_heads = 2. 
- we get the d_out from the final output (context_vector: which will be a 3 X 6 matrix) --> 3 tokens, and 6 vectors for each token
- Get Head_dimension (head_dim) = d_out / num_heads ==> 6/2 = 3
- GPTs use up to 96 attention heads, so num_head = 96 and input dimensions (vector embedding) will be large too for each token.

3. Initialize the trainable weight matrices for Key, Query and Value ---> Wq, Wk, Wv = d_in * d_out (6 X 6 matrix tensors). d_in = d_out is needed for easy matrix multiplication of the vector embedding with each of the trainable weight matrices.

4. Calculate Keys, Queries and Values (Keys = Input * Wk, Queries = Input * Wq, Values = Input * Wv). Mathematically, (1 X 3 X 6 matrix * 6 X 6 matrix tensors = 1 X 3 X 6 matrix) should be the output after multiplication. where 1=batch_size(b), 3=num_tokens, 6=d_out (output dimensions) = In matrix form, (b, num_tokens, d_out)

5. Unroll last dimension of Key, Query and Values matrices to include num_heads (number of heads) and Head_dimension (head_dim). This will give 4-dimension matrix
- head_dim = d_out / num_heads ==> 6/2 = 3
- (b, num_tokens, d_out) = (b, num_tokens, num_heads, head_dim) ---> (1,3,6) matrix = (1,3,2,3) matrix. 
- In elaboration, 1=batch size, 3=number of token, 2=each token with 2 heads, 3=each head with 3 head dimensions

6. Group each of Key, Query and Values matrices by the num_heads instead of num_tokens using transpose.
- (b, num_tokens, num_heads, head_dim) = (b, num_heads, num_tokens, head_dim)  ----> (1,3,2,3) matrix = (1,2,3,3) matrix.
- The grouping of num_head (number of head) is necessary to compute attention scores for each Head group separately.

7. Get the Attention Scores --> Queries * Keys.Transpose(2,3) 
- in Matrix form: (b, num_heads, num_tokens, head_dim) * (b, num_heads, head_dim, num_tokens) = (b, num_head, num_tokens, num_tokens) which is the attention scores
- (1,2,3,3) * (1,2,3,3) = (1,2,3,3)

8. Get Attention Weights: (1,2,3,3) = (b, num_head, num_tokens, num_tokens)  same as attention scores
- Mask Attention Scores to implement Causal Attention (mask each word after a particular token as -inf)
- Get the softmax ---> Necessary to normalize the -inf to zero and ensure each row adds up to 1
- We divide by the square root of head_dim == square root of (d_out / num_heads)  == square root of (6/2). This is for back propagation and stable variance
- Also, implement drop out

9. Compute Context Vector = Attention Weights * Values
- (b, num_heads, num_tokens, num_tokens) * (b, num_heads, num_tokens, head_dim) = (b, num_heads, num_tokens, head_dim)
- In Matrix form, (1,2,3,3) * (1,2,3,3) = (1,2,3,3)

10. Reformat Context Vectors using Transpose. This is necessary to merge the number of heads and head_dimension
- (b, num_heads, num_tokens, head_dim) ==> (b, num_tokens, num_heads, head_dim)
- In Matrix form, (1,2,3,3) ==> (1,3,2,3)

11. Merge the num_heads and head_dim in the Context Vector. This is called Flatten  
- (b, num_tokens, num_heads, head_dim) ==> (b, num_tokens, d_out)
- In Matrix form, (1,3,2,3) ==> (1,3,6)

- Note: we use 1 batch here but more batches are used in training GPT models

* Layer Normalization from Transformer Block:

- Training deep neural network with many layers can be challenging due to vanishing/exploding gradients and unstable training dynamics.

- Layer Normalization improves stability and efficiency of neural network training

- Back Propagation of layers depends on the Gradient. The gradient must not be too large or too small, hence there will be explosion and vanished gradients. Gradients are gotten by multiplication of outputs from input layers, to outputs from hidden layers during Back Propagation. Note that outputs from input layer acts as inputs to the hidden layers.

* Goal: the goal is to avoid explosion of gradients if outputs from each layer is large OR vanishing of gradients if output from each layer is small.

- Layer Normalization helps keep the Gradient stable.

- Layer Normalization helps to keep the inputs from every layer fixed to avoid internal covariate shift (delay of convergence)

- Layer Normalization adjusts the outputs of layers (every input embedding) to have mean = zero and variance = 1 to speed up un convergence.

Lecture 21: GELU Activation Function in the LLM Architecture - GPT ARCHITECTURE PART 3  (LLM Architecture.ipynb notebook)

* GELU activation function lives inside the Feed Forward Layer in the Transformer Block

* GELU Activation Function is used in GPTs inplace of RELU activation:

- if X = +ve value (X > 0), RELU outputs the value and if X = -ve value (X < 0), RELU outputs zero. This sometimes leads to dead neurons because of the zero.

- Meanwhile, GELU activation function doesn not output zero for any -ve X value (X < 0). This solves the dead neuron problem (the neuron still learns during training)

- GELU is better than RELU

* Default GELU Activation FUnction used in Training GPTs:

- GELU(x) == 0.5 * X * (1 + tanh[sqrt(2/π) * (X + 0.044715 * X**3)]   ---> it's in the notebook as code with torch

- Multiply the 768 dimension by 4 of the input in the Linear layer 1 before it goes into the GELU Activation Function layer. The output dimension from GELU are now compressed back to 768 dimensions initially from the input.

Lecture 24: CODING THE 124-million PARAMETER GPT-2 MODEL (LLM Architecture.ipynb notebook)

* Goal: Assemble a fully working version of the original 124-million parameters GPT-2. 

* Procedural Steps:

1. We'll take input texts and pass them through the entire GPT-2 Architecture.

2. Convert the input texts into tokens using the BYTE PAIR Encoder (sub-word tokenizer) and get the token IDs of all the input texts from the GPT-2 vocabulary (50257)

3. Convert the each token ID into token embedding vector --> GPT-2 has 768 embedding dimensions (size)

4. Get the position embedding vector for each token. --> The first token will be Position 1, alongside the 768-embedding dimensions(size). The total position is also the context size (number of tokens needed to predict the next token).

* Note: similar to token embeddings and positional embeddings, we do not know the embedding values for each token because they will be trianed.

5. Add the token embeddings and positional embeddings for each token to get input embeddings with embedding size 768. Input Embedding = Token Embedding + Positional Embedding

6. Apply Dropout by turning off some 50% of each input embeddings (elements) of size 768-dimensions to zero. --> 50% of size 768 embeddings. this prevents overfitting and improves generalization when some neurons are being lazy to optimize.

7. Input embeddings with dropout enters the Transformer Block while also preserving each input 768 dimensions

- Layer Normalization 1: the MEAN = 0 and VARIANCE = 1 for every input embedding with 768 dimensions

- Masked Multi-Head Attention: convert the input embeddings vectors to context vectors using Query, Key and Value Matrices. Context vector captures the attention/meaning of each 'query' vector with respect to other vectors. Embedding size still 768 dimensions. This is the most import stepRefer to video for more knowledge on this. 

- Dropout: turns 50% of each context vectors to zero. Same as previous dropout step.

- Shortcut Connections: Add the output of dropout layer to the input layer of the previous Layer Normalization 1 to provide another means for gradient flow and solve the problem of vanishing gradient.

- Layer Normalization 2: the MEAN = 0 and VARIANCE = 1 for every context embedding vector with 768 dimensions

- Feed Forward Block: Input Linear Layer: expansion of each context vector by 4. 
Summarize below:
Multiply the 768 dimension by 4 of each input in the Linear layer 1 before it goes into the GELU Activation Function layer. The output dimension from GELU are now compressed back to 768 dimensions initially from the input.

- Dropout: turns 50% of each context vectors to zero. Same as previous dropout step.

- Shortcut Connections layer: Add the output of dropout layer to the input layer of the previous Layer Normalization 2 to provide another means for gradient flow and solve the problem of vanishing gradient. This is the output of the Transformer Block

- Exit of the Transformer Block

8. Pass the output from Transformer block into a final layer normalization block 

9. Pass the output from final Layer Normalization block into an Output Neural Network Layer. Mathematically, (4 X 768 dimensions * 50257 X 768 dimensions) to get the Logits (number of tokens * 50257 vocabulary size). In this case: (4 * 502570) 

- If batch side is considered, Logits = Batch size * Number of Tokens * Vocabulary Size

- Logits: for every input token, we look at the 50257 vocabulary size to get the token with highest dimension/value. The highest dimension/value token will be the predicted next token. Then, we perform Autoregressive method where the output tokens serve as input to get the next predicted token.

- Here the column dimensions change from 768 to 50257 columns  

- In summary, GPT model outputs tensors with shape [batch-size, num_tokens, vocab_size]

* Context Size: how many input words we look at before predicting the next word (just for definiton purposes)

Lecture 26: Measuring the LLM Loss Function  stage2_TrainingLLM\1.LLM Loss function.ipynb)

* Goal: get the closeness of our predicted token IDs to the target token IDs by minimizing the loss function during training.

- Loss Function tell us how good or bad the LLM performed. if the LLM performs bad, we can minimize the loss, which will aid gradient descent and back propagation.

- Loss function compares the Target and Predicted Values to see how close they are. We use the input values to get the predicted values (predicted token IDs), why we know the target values (target token IDs)

* Note: number of rows == number of batches

* Cross Entropy Loss: measures difference between 2 probabilities distribution . It should be brought down close to zero as much as possible

- Get the probabilities of predicted logits from the model with input tokens IDs. Flatten the logits so they merge together.

- Merge both batches the target token into a flatten matrix, Convert the target token IDs into probabilities, take the logarithms of each target token, get the mean of the logarithm and take the negative of the mean. This is done so each probability of the merged target token and the merged logit output is close to 1.

- "torch.nn.functional.cross_entropy" line of code does all the above processes for us

* Perplexity: Another measure of loss like cross entropy 

- This measures how well the distribution predicted by the model matches the actual distribution of words by the dataset. More interpretable way of mudel uncertainty in predicting next token.

- Lower Perplexity Score = better predictions. 

- Mathematically, Perplexity = torch.exp(loss) --> the gives the number of tokens to be predicted from out of the total number of vocabulary size (50257)

Lecture 29: Temperature Scaling in Large Language Model (LLMs) (stage2_TrainingLLM\3.LLM Pretraining.ipynb)

- Here, we do not pick the logit token with the highest probability. Instead, we sample next all output tokens according to probability score. --> Replace argmax with multinomial probability distribution in the code.

- Temperature changes the distribution of probabilities. 

1. Get the logits

2. Scale logits to temperature by dividing each logits by selected temperature

3. Apply Softmax on the scaled logits to get a tensor of probabilites 

4. Then sample the probabilities to get multinomial distribution.

- If scaled logit with higher temperature, entropy increases and every logit has the probability of being the next token. If scaled with low temp, entropy becomes low and there is always a fixed output logit (same as using argmax to get the highest probability). 

- Be careful to choose a balanced temperature for controlling the entropy and randomness of the next predicted token.

Lecture 30: Top-K Sampling in Large Language Model (LLMs) (stage2_TrainingLLM\4.LLM Decoding Strategies.ipynb)

* Goal: 

- Top- K Sampling restricts the sampled tokens to the TOP-K most likely tokens and exclude all other tokens.

1. Get the output Logits

2. Perform Top-K sampling to get the highest logits tensors (here, we use k=3). Other non-selected logits will be -inf(infinity) and applying SoftMax will make them zero.

3. Scale logits to temperature by dividing each Top-K logits by selected temperature.

4. Apply Softmax on the scaled Top-K logits to get a tensor of probabilites. Here, the -inf tenosors will be zero.

5. Then sample the probabilities to get multinomial distribution.

* Note: Multimonial Distribution prevents overfitting

Lecture 33: Introduction To LLM Finetuning -- Python Coding with Hands-On Examples (stage3_Finetuning\1.Classification finetuning data loading.ipynb)

* Finetuning: Adapting the pretrained model to a specific task by training the model on additional domain data. During finetuning process, the pretrained model weights and parameters change inorder to adapt to the new dataset.

* 2 Types of Finetuning:

- Instruction FineTuning: training an LLM on a set of tasks using specific instructions. 

- Classification FineTuning: model is trained to a specific set of labels such as spam or no spam.

- Parameter Efficient FineTuning (PEFT): this is a form of instruction finetuning that is more efficient than full finetuning. PEFT addresses this by updating only a subset of parameters, effectively freezing the rest. This reduces the number of trainable parameters, making memory requirements more manageable and forgetting catastrophic forgetting.

- Various ways of achieving Parameter Efficient FineTuning:

1. LORA (LOW RANK ADAPTERS) - instead of finetuning all the weights matrices of the pretrained LLM, two smaller matrices that approximate the larger matrix are finetuned. The smaller matrices constitute the finetuned LORA Adapters and are loaded into the pretrained model.

2. QLORA (QUANTIZED LOW RANK ADAPTERS) - this is a more efficient iteration of LORA. QLORA takes a step further by quantizing the weights of Low Rank Adapters (smaller matrices) to lower precision. eg 4-bits instead of 8-bits

Lecture 2: Chain of Thought Reasoning (Few-Shot CoT & Zero-Shot CoT)

* Method 1- Inference Time-Compute Scaling: 
Allocating more test-time compute resources to an LLM to impact its reasoning and generate better answers. This is the same as humans thinking more on a problem using our mental energy to achieve better outcomes.

You only prompt the LLM once from an input/output pair (examples), and the LLM generates outputs (answers) in a similar way for any other input examples it sees. - If no input/output pairs are provided, the prompting is Zero-Shot Prompting. And if a reasoning step is included in the prompt, it's called Zero-Shot Chain of Thoughts prompting

Lecture 3: Verifiers (Beam Search)

- You ask the LLM a question, the LLM generates different answers. The LLM reasoning layers reasons through the generated answers, and the the verification layer verifies all the generated answers before producing the best answer.

- Verification increase the inference test time compute of the LLM

- Verification can be done by either human or models. If Verification is done by model, the model is called Reward Model.

* 2 types of Reward MOdels:
- Outcome Reward Model (ORM) --> gives a reward by scoring only the final outcome of the reasoning model.
- Process Reward Model (PRM) --> indivdual reasoning/thinking step of the Reasoning Model is scored.

* Types of Verifiers:

1. Majority Voting  --> the verifier assesses the majority of answers produced by the reasoning model. There is no verification layer here and the majority answer could be wrong.

2. Best of N --> N samples are produced by the LLM, and the verifier (ORM) chooses the best sample of all produced samples. Also, if verifier is PRM, it scores each thinking step produced by the reasoning model.

3. Beam Search --> this verifier (PRM) combines with reasoning model to explore multiple reasoning paths before arriving at the final answer. 
Two parameters here: 
- number of beams parameter (different thoughts generated by the model)
- beam width parameter (selected thoughts from generated thoughts) and

Lecture 8: Topic Modelling Using BERTopic

* Here, we assign labels (keywords) to each cluster gotten from the previous lecture using the same code.

* Latent Dirichet Allocation: a traditional approach that uses bag of words to extract keywords and do not take into account context or meaning.

* BERTopic: modular topic modelling framework that leverage clusters of semantic similar text to extract various types of topic representations.

- BERTopic Modularity ---> embedding model (SBERT) > dimensionality reduction (UMAP) > Clustering (HDBSCAN) > CountVectorizer (get term frequency) > TF-IDF

- BERTopic approach is called the Bag-of-Words approach.

* Term Frequency * Inverse Document Frequency (TF-IDF) will be used to put more weights on words that are meaningful to the cluster, and put less weights on words used all across the clusters

- Term Frequency (Count Vectorizer) is the number of times a word appears in a cluster.

- Inverse Document Frequency is frequency of all documents across all clusters

Mathematically, TF-IDF_word_score = log[(Average Frequency of all words across all clusters / Total Frequency of each word in a cluster ) + 1]  OR

Alternatively, IDF = log(n/df) where n will represent the total number of clusters/documents and df will represent the number of clusters/documents in which the word has appeared in.

- The TF-IDF score gives which words should be used as keywords to describe each cluster.

Lecture 9: LLM (Representative and Generative Models) for Text Clustering and Topic Modelling (BERTopic)

* BERTopic (Bag of Words) approach limitation: it does not take semantic structures into account when representing a topic.

* Bag of Words approach is used to get the keywords for each cluster, but combined with embedding (representation) models to rerank the keywords for semantic meaning of the keywords to each cluster documents. This procedure is called FineTune Representation or Rerank Representation

* Different Finetune Representative Blocks
1. KeyBERT
2. MMR (Maximum Marginal Relevance)
3. Spacy

* How the KeyBERT Representative Model Works: 
- the representative model embeds documents in a clusters
- the keywords got from TF-IDF are also embedded using an pretrained embedding model
- get the Cosine Similarity Score between the embedded keywords and the embedded documents in the cluster(s). Reranking gives from Highest Similarity Score to Lowest similarity score
- KeyBERT doesn't account for abbreviations in a cluster of documents, even if the abbreviations are relevant.

* How the MMR Representative Model Works:
- filter redundant keywords (words that appear often) and keep those that contribute something new to topic representation.
- diversification of topic representation --> finds Keywords that are diverse but still related to compared documents in the cluster.
- specify a diversity parameter.

* Note: KeyBERT and MMR can be used/stacked together or independently

* Generative models look at 3 documents in a cluster and its corresponding Keywords to generate a label.

1. Specifity - be specific in the prompt about what's necessary
2. Hallucination - prompt the LLm to include 'I don't Know' if it dies not know the answer so it does not hallucinate.
3. Order - consider Primacy Effect (instruction at the beginning) or Recency Effect (instructions at the ending) when prompting because LLMs focus more on information at the beginning and ending of prompts.

Lecture 14: LLM Quantization

* Quantization is the compression of original LLM into a smaller version of the model (GGUF format). It reduces the number of bits (memory) needed to represent LLM parameters (weights) while attempting to retain most of the information.

- VRAM (Video Random Access Memory): to run large memory LLMs, and can take small memory of quantized LLMs

- Float 16-bit (FP16) has 3 parts:
1. Sign (1-bit)
2. Exponent (5-bits)
3. Significand/Mantissa (10-bits)
 
- Float 32-bits (FP32) representation has higher precision than Float 16-bits (FP16). Also has sign, exponents and Significand(Mantissa) as FP16. Most LLMs weights are represented in 32-bits (full precision)

- More bits = More Precision = More memory consumption.

- Mathematically: LLM Memory = [number of bits / 8] * number of parameters. 

- For 64-bits model weights representation and 70-billion parameters, llm-memory = 64/8 * 70 ==> 560GB memory size

- For 32-bits model weights representation and 70-billion parameters, llm-memory = 32/8 * 70 ==> 280GB memory size

- For 16-bits model weights representation and 70-billion parameters, llm-memory = 16/8 * 70 ==> 140GB memory size

- Tradeoff: low memory is directly proportional to low precision.

- Quantization and Dequantization involves loss of information

- GGUF is file format for using quantized model on a CPU with low VRAM. Commnly used with `llamacpp`

Lecture 26: Vision Transformers - How and Why they Work

* Models which can handle both texts and images are called Multimodal. Also videos, audio models are termed Multimodal models.

# Critical Steps of Processing an Image through a Vision Transformer (similar to text data for GPT-2 Architecture)
1. Convert image into tokens
2. Convert tokens into token embedding vectors
3. Add positional embedding vectors to token embedding vectors to get input embeddings.

* Converting images to tokens involves dividing the image into 16-patches (tokens) of images (sub images) (pixels-16X16). Each patch is flattened out to get both token and positional embedding vectors and sum them together to get input embeddings (embedded patches). The embedded patches, similar to text is then fed into the transformer block. This is the priciple behind Vision Transformer.

* Principle of Vision Transformer: An Image is worth 16 X 16 words

* Linear Projection of Flattened Patches is used to convert the flattened patches into vectors. Each patch can be termed as a token.

* Traditional way of working with image classification is CNN (Convolution Neural Network), which involves extracting feature maps, and stacking of layers (filtering layer, max pooling layer

* Vision Transformer does not rely on translational equivariance and localiation, which CNN relies on.

* CNN outperforms ViT on small dataset, while Vision transformers outperm CNN on large datasets

Lecture 27: Introduction to CLIP (Multimodal Embedding Models)

* Multimodal Embedding Models have embeddings of both image and text in the same vector space
* Multimodal embeddings models create embeddings for multiple modalities in the same vector space.

- CLIP (Contrastive Language Image Pre-training): a multimodal embedding model that computes embeddings of both images and text. Resulting embeddings of both texts and images lie in the same vector space, which means image embeddings can be compared to text embeddings.

- The embeddings of images and texts are computed using the Vision Transformer for Images and Text transformer for texts, before projecting both embeddings in a bigger vector space.

* Contrastive Learning: when you feed the multimodal model with new batches of images and texts to update its weight for training. These new batches are fed into the model after the cosine similarity between an image and text is found.

* Clip does not generate new texts or images, just embeds images with its corresponding texts and capture semantic meaning between them.

The basic idea behind transformers, such as those used in ChatGPT, is to split a sentence into words (or tokens) and then convert these tokens into vector representations. Those vector representations are used to predict the next word. Vision transformers are very similar in many aspects. In vision transformers, you divide an image into sub-images (also called patches). Then you create vector representations of each of these patches and add an additional vector called CLS token and pass them through something called multi-head attention (which is very similar to the attention used in transformers for text-based data). Finally, you use CLS token from the input for performing classification.

In this lecture, we code a vision transformer completely from scratch using PyTorch. We divide the transformer into three parts.

Part one is the embedding where the image is split into patches. Patches are converted into vector embeddings on top of these vector embeddings, at the beginning, we concatenate a CLS token and all of these tokens are added with the position embedding. 

Part two is the transformer encoder which repeats itself in series. So we can define four transformer encoders in series. We can just define one transformer encoder. It depends on what we want to do. Inside this transformer encoder: - There are two layer normalizations - There are two places where there are residual connections which help us minimize vanishing gradient - There is also one multi-head attention Within the multi-head attention there are parallel layers of attention which process only a fraction of the dimension of the input patch embeddings.  At the output of the multi-head attention, these partial vectors are concatenated back to get the original input embedding dimension, and then these vectors are passed through a multi-layer perceptron to increase the representation capability. 

Part three is the multi-layer perceptron (MLP) head, which performs the classification. Unlike text-based transformers, we are not going to predict the next batch. Typically, transformers perform next word prediction, but here we are trying to classify the image into one of the 10 or 5 (how many of our number of classes we want) classes we are using. For performing the classification, we only use the CLS tokens. So only the CLS token is passed through the MLP head for classification. 

In this lecture, we build a vision transformer from scratch using PyTorch. We don't use any pre-trained model. We train the built vision transformer from scratch, and for the dataset, we use the famous MNIST dataset so that we are performing a 10-class classification. We get 96%+ training accuracy and similar validation accuracy which is on par with what many CNN's can do. With further hyperparameter tuning, we can even push this to 99% accuracy or beyond.

Lecture 26: Yolo Introduction (YOLO - You Only Look Once) 

* YOLO improved on models like R-CNN, Faster R-CNN which look at an image multiple times, and draw many bounding boxes before detecting the image. Yolo looks at the image just once for detection.

* Yolo involves dividing the image into matrix grid cells inorder to apply bounding boxes and confidence score for the output.

* Confidence_score = intersection of image prediction * Probability of Class object. Mathematically, confi_score = image score * class object probability.
